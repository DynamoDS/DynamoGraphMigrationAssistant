Dynamo Graph Migration Assistant

The Dynamo Graph Migration Assistant automates several tasks that are needed to update a graph to a newer Dynamo version. You can migrate a single graph or several graphs at once to one of the target Dynamo versions offered. Several settings are available to customize the graph update.

Note that you cannot use this tool to migrate graphs to an older version. Any graphs already in the target version will be copied to the target folder with no changes.

Migrating graphs

  1. To prepare for migrating your graphs, place all the graphs you want to migrate into a folder. This folder can have subfolders.
  2. In the Dynamo Graph Migration Assistant, select a source folder. This is the folder from step 1. The number of Dynamo graphs to be migrated appears next to the Export button. The names of the graphs are shown under the Graphs to Migrate header.
  3. Select a target folder. When you begin the migration process, migrated versions of your graphs will be created in the target folder. The original graphs will remain in the source folder.
  4. Select a target Dynamo version. This needs to be newer than the version of the graphs in the source folder.
  5. Select options under General Settings. More information about each setting is available in the Settings section below.
  6. Select options under Recommended Graph Updates.
  7. Select options under Optional Graph Updates.
  8. Click Export to begin the migration process. You can keep track of progress in the Graphs to Migrate section. Checkmarks appear next to the name of each migrated graph.
  9. You will receive a confirmation message once the migration is complete. Migrated graphs will be available in the target folder.

Settings

You can customize the migration process using the settings provided.

General Settings

Keep folder structure: Choose if you want to duplicate the source folder’s structure in the target folder. If you do not check the Keep Folder Structure box, migrated graphs will be placed in the root target folder without subfolders.

Resume previous session: If you previously interrupted a migration process, you can resume it by checking the box. Progress is based on a log.txt file created in the target folder, so be sure to choose the same target folder that you used for the migration process you want to resume.

Recommended Graph Updates

These settings improve the migrated graphs’ usability and functionality and are recommended based on your selected target version.

Cleanup node layout: This setting moves nodes in the workspace to prevent them from overlapping.
Update If nodes: This setting replaces outdated If nodes with updated versions.

Optional Graph Updates

These settings may be useful based on your graph practices.

Force input order: Creates an input order (for example, to use in Dynamo Player) by appending a prefix to the name of each node marked as “Is Input.” The prefix is 001|, 002|, etc. Numbering will begin with the node closest to the top of the workspace.
Swap line breaks to pinned notes: If a node name includes a line break, this option will take the text before the line break and turn it into a pinned note. The text after the line break will remain as the node’s name.
